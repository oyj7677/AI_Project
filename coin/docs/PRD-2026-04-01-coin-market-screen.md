# Summary

`coin/web`의 첫 제품 화면으로, 사용자가 KRW 마켓 코인 목록과 선택 종목의 가격 그래프를 한 화면에서 볼 수 있는 웹 대시보드를 정의한다.

# Problem

- 현재 사용자는 거래소 앱과 여러 탭을 오가며 종목 목록, 현재가, 차트를 따로 확인해야 한다.
- 이후 자동매매 기능을 붙이려면 먼저 시장 상태를 빠르게 읽는 기본 화면이 필요하다.
- 자동매매 이전 단계에서도 사람이 수동으로 시장을 점검할 수 있는 베이스 UI가 있어야 한다.

# Goals

- 사용자가 대표 코인 종목 목록을 빠르게 훑을 수 있어야 한다.
- 사용자가 선택한 종목의 가격 흐름을 그래프로 즉시 볼 수 있어야 한다.
- 이 화면이 이후 자동매매 전략, 시그널, 포지션 화면으로 이어지는 첫 진입점이 되어야 한다.

# Non-Goals

- 실제 주문 실행과 자동매매 전략 실행은 이번 범위에 포함하지 않는다.
- 로그인, 사용자 계정, API 키 관리도 이번 범위에 포함하지 않는다.
- 복잡한 보조지표 세트와 백테스트 기능은 1차 범위에 포함하지 않는다.

# Users / Scenarios

## Primary User

- 주 사용자: 개인 개발자이자 초기 운영자
- 사용 목적: 시장을 빠르게 확인하고 다음 자동매매 기능 설계의 기준 화면으로 사용

## Core Scenarios

- 사용자가 KRW 코인 목록을 훑는다.
- 사용자가 특정 코인을 선택한다.
- 사용자가 시간 범위를 바꿔가며 가격 흐름을 본다.
- 사용자가 검색을 통해 원하는 종목을 빠르게 찾는다.

# Scope

## Included

- KRW 마켓 코인 목록
- 종목 검색
- 선택 종목 현재가/등락 정보
- 선택 종목 가격 그래프
- 시간 범위 전환
- 기본 로딩, 오류, 빈 상태
- 모바일/데스크톱 반응형 지원

## Excluded

- 주문, 포지션, 자동매매 전략 실행
- 사용자 인증
- 실시간 WebSocket 스트리밍 고도화
- 관심 종목 저장

# Information To Show

## Coin List

- 코인명
- 마켓 심볼
- 현재가
- 전일 대비 등락률
- 24시간 거래대금

## Chart Area

- 선택 종목명
- 선택 시간 범위
- 가격 라인 차트
- 고점/저점
- 마지막 가격

# UX Requirements

- 데스크톱에서는 종목 목록과 차트를 동시에 볼 수 있어야 한다.
- 모바일에서는 목록과 차트를 좁은 폭에서도 읽기 쉽게 재배치해야 한다.
- 기본 선택 종목은 대표성이 높은 `KRW-BTC`여야 한다.
- 종목 선택 후 차트 갱신은 1초 내 체감 반응을 목표로 한다.

# Success Metrics

- 초기 데이터 로딩 3초 이내
- 종목 전환 후 차트 갱신 1초 이내
- 대표 종목 검색 및 선택 10초 이내
- 내부 검증 기준으로 "시장 파악용 첫 화면으로 충분하다"는 평가 확보

# Acceptance Criteria

- [ ] 사용자가 KRW 마켓 코인 목록을 볼 수 있다.
- [ ] 사용자가 검색으로 종목을 필터링할 수 있다.
- [ ] 사용자가 종목을 선택하면 차트와 요약 정보가 바뀐다.
- [ ] 사용자가 시간 범위를 바꾸면 그래프가 갱신된다.
- [ ] 로딩 및 오류 상태가 사용자에게 명확하게 보인다.

# Risks

- Upbit 공개 API rate limit 또는 응답 지연
- 종목 수가 많아질 때 모바일 리스트 사용성 저하
- 차트 표현이 너무 단순해 보일 위험
- 자동매매 기능 기대와 현재 범위의 간극

# Open Questions

- 업비트 단일 거래소로 시작할지 멀티 거래소를 염두에 둘지
- 실시간 업데이트를 1차에 넣을지 후속 단계로 미룰지
- 기본 시간 범위를 `1D`로 둘지 `1H`로 둘지
- 정렬 기준을 거래대금 우선으로 고정할지 옵션화할지

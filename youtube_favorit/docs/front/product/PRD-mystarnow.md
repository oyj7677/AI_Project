# Summary

1차는 YouTube와 Instagram에 흩어진 인플루언서 소식을 한곳에서 모아 보여주는 서비스의 초기 기획 문서다. 사용자는 한 명의 인플루언서 기준으로 공식 채널, 현재 라이브 여부, 최근 업로드, 최근 SNS 게시물, 예정 방송을 빠르게 확인할 수 있어야 한다. X, 치지직, 숲은 후속 확장 대상으로 남긴다.

# Problem

- 팬은 인플루언서 한 명을 챙기기 위해 유튜브와 인스타그램을 계속 오가야 한다.
- "지금 방송 중인지", "최근에 무엇을 올렸는지", "어느 플랫폼이 메인인지"를 한 번에 보기 어렵다.
- 신규 유입 사용자는 이름은 알지만 공식 계정들을 모두 찾기 어렵다.
- 플랫폼마다 콘텐츠 형식이 달라 활동 흐름을 종합적으로 파악하기 힘들다.

# Goals

- 인플루언서별 공식 플랫폼 링크를 한 화면에서 제공한다.
- 현재 라이브 여부를 가장 먼저 보여준다.
- 최근 영상, 게시물, 짧은 소식, 예정 방송을 묶어 보여준다.
- 1차 지원 플랫폼은 `YouTube`, `Instagram`으로 제한한다.
- 팬이 특정 인플루언서의 최신 활동을 10초 안에 파악할 수 있게 한다.
- 초기에는 "정보를 찾기 쉬운 허브" 역할에 집중한다.

# Non-Goals

- 1차 범위에서 자체 영상 재생 또는 채팅 기능은 만들지 않는다.
- 1차 범위에서 모든 인플루언서를 자동 수집하는 대규모 크롤링 서비스는 목표로 하지 않는다.
- 1차 범위에서 `X`, `CHZZK`, `SOOP` 연동은 포함하지 않는다.
- 1차 범위에서 커뮤니티, 댓글, 후원, DM 기능은 제외한다.
- 1차 범위에서 추천 알고리즘 중심 서비스로 확장하지 않는다.

# Product Concept

## One-line Definition

인플루언서별 모든 소식을 모아보는 개인 방송 허브 서비스

## Why It Matters

- 사용자는 "어디 들어가야 하지?"보다 "지금 뭐 했지?"를 알고 싶어한다.
- 링크 모음 수준을 넘어서 라이브, 최근 콘텐츠, 최근 SNS 활동을 함께 보여줘야 재방문 가치가 생긴다.
- 인플루언서 단위 구조를 먼저 잡아두면 이후 알림, 즐겨찾기, 카테고리 추천, 랭킹으로 확장하기 쉽다.

# Target Users

## Primary Users

- 특정 인플루언서를 꾸준히 챙겨보는 팬
- 최신 소식을 빠르게 확인하려는 사용자
- 여러 플랫폼을 따로 확인하는 것이 번거로운 사용자

## Secondary Users

- 인플루언서 매니저 또는 운영 담당자
- 팬 페이지, 클립 페이지 운영자

# Key User Scenarios

## Scenario 1. 인플루언서 검색

- 사용자가 인플루언서 이름을 검색한다.
- 서비스는 대표 프로필과 공식 계정 링크를 보여준다.
- 사용자는 한 번에 유튜브, 인스타그램, X, 치지직, 숲 계정을 찾는다.

## Scenario 2. 지금 방송 중인지 확인

- 사용자가 홈에 들어온다.
- 서비스는 현재 라이브 중인 인플루언서를 상단에 노출한다.
- 사용자는 플랫폼 버튼을 눌러 바로 해당 방송으로 이동한다.

## Scenario 3. 최근 활동 확인

- 사용자가 인플루언서 상세 페이지에 들어간다.
- 서비스는 최근 유튜브 업로드, 인스타그램 게시물, X 포스트, 라이브 상태를 시간순으로 보여준다.
- 사용자는 최근 활동 흐름을 빠르게 파악한다.

## Scenario 4. 즐겨찾기 관리

- 사용자가 자주 보는 인플루언서를 즐겨찾기에 추가한다.
- 서비스는 홈에서 즐겨찾기 인플루언서만 모아 보여준다.

# Core Value Proposition

- 인플루언서 한 명의 여러 플랫폼 소식을 하나로 묶어준다.
- 팬이 가장 궁금한 정보인 라이브 상태와 최신 업로드를 즉시 보여준다.
- 신규 팬도 공식 계정을 쉽게 찾을 수 있게 한다.

# Feature Proposal

## 1. 인플루언서 디렉터리

설명:
인플루언서를 검색하고 발견하는 기본 기능이다.

포함 정보:

- 활동명
- 프로필 이미지
- 한 줄 소개
- 주 활동 카테고리
- 지원 플랫폼 배지
- 현재 라이브 여부

예시:

- "하루 | 게임, 토크 중심 스트리머 | YouTube, Instagram, X, CHZZK"

## 2. 공식 채널 허브

설명:
인플루언서 상세 페이지에서 플랫폼별 공식 채널을 카드 또는 버튼으로 정리한다.

채널 카드 예시 항목:

- 플랫폼명
- 계정 핸들
- 공식 여부
- 최근 업데이트 시간
- 바로가기 버튼

지원 플랫폼 예시:

- YouTube
- Instagram
- X
- CHZZK
- SOOP

## 3. 라이브 상태 집계

설명:
홈과 상세 페이지 모두에서 가장 중요한 기능이다.

표시 방식:

- `LIVE NOW` 배지
- 플랫폼명
- 방송 제목
- 시작 시간
- 시청 링크 버튼

예시:

- "하루 - 치지직에서 20:10부터 방송 중"
- "민도 - 숲에서 21:00 예정"

## 4. 최근 콘텐츠 피드

설명:
영상과 SNS 활동을 함께 모아 최신성을 보여준다.

콘텐츠 종류:

- 유튜브 영상
- 유튜브 쇼츠
- 인스타그램 게시물
- 인스타그램 릴스
- X 포스트
- 방송 공지

## 5. 오늘 예정 방송

설명:
사전 공지나 운영자 입력 기반으로 예정 방송을 정리한다.

MVP에서는 다음 중 하나로 시작:

- 운영자가 직접 예정 방송을 등록
- 인플루언서가 올린 공지를 수동 반영

## 6. 즐겨찾기

설명:
로그인 없이도 자주 보는 인플루언서를 모아볼 수 있게 한다.

MVP 방식:

- 로컬 스토리지 기반 즐겨찾기

## 7. 정보 수정 요청

설명:
계정 변경, 플랫폼 추가, 비공식 링크 오류에 대응하기 위한 운영 장치다.

MVP 방식:

- "정보 수정 요청" 버튼
- 구글 폼 또는 이메일 제보

# Data Collection Strategy

## MVP

- 운영자가 인플루언서와 공식 채널 정보를 수동 등록한다.
- 플랫폼별 최신 콘텐츠는 최근 1~5개 정도만 표시한다.
- 라이브 상태는 연동 가능한 플랫폼부터 우선 지원한다.

## Phase 2

- 제보 기반으로 링크 수정 및 신규 인플루언서 등록을 받는다.
- 주기적인 배치 작업으로 라이브 상태와 최신 콘텐츠를 갱신한다.
- 플랫폼별 데이터 수집 안정성을 높인다.

## Why Start Manual

- 초기에 가장 중요한 것은 데이터 자동화보다 공식 링크 정확도다.
- 소수의 인플루언서를 먼저 정교하게 관리하는 편이 사용자 신뢰 확보에 유리하다.

# Recommended Launch Strategy

## Version 1

- 인플루언서 20~50명 수동 등록
- YouTube, Instagram, X, CHZZK, SOOP 지원
- 모바일 웹 우선 출시

## Version 2

- 라이브 시작 알림
- 카테고리별 탐색
- 최근 활동량 기반 정렬

## Version 3

- 개인화 피드
- 실시간 인기 순위
- 클립/하이라이트 큐레이션

# Success Metrics

- 사용자가 특정 인플루언서의 공식 채널을 30초 안에 찾을 수 있다.
- 사용자가 홈에서 라이브 여부를 5초 안에 확인할 수 있다.
- 즐겨찾기 등록 전환율 20% 이상
- 상세 페이지 체류 시간 1분 이상
- 정보 수정 요청 반영률 80% 이상

# Acceptance Criteria

- [ ] 사용자가 인플루언서 이름으로 검색할 수 있다.
- [ ] 사용자가 상세 페이지에서 플랫폼별 공식 링크를 볼 수 있다.
- [ ] 사용자가 현재 라이브 여부를 확인할 수 있다.
- [ ] 사용자가 최근 영상과 최근 SNS 소식을 함께 볼 수 있다.
- [ ] 사용자가 즐겨찾기에 인플루언서를 저장할 수 있다.
- [ ] 운영자가 잘못된 링크를 수정할 수 있는 최소 운영 흐름이 있다.

# Risks

- 플랫폼별 API 정책 차이로 데이터 수집 난이도가 다르다.
- 공식 계정 판별 기준이 불명확하면 신뢰도가 떨어질 수 있다.
- 라이브 상태와 최근 게시물 갱신 주기에 따라 정보 신선도가 달라진다.
- 너무 많은 플랫폼을 초기에 붙이면 운영 비용이 커질 수 있다.

# Open Questions

- 1차에서 어떤 플랫폼부터 안정적으로 지원할지
- X와 인스타그램의 데이터 노출 범위를 어디까지 가져올지
- 라이브 상태 갱신 주기를 몇 분으로 잡을지
- 운영자 수동 등록 화면이 필요한지
- 로그인 없이도 충분한지

# Recommended Next Steps

1. 1차 지원 플랫폼 우선순위를 확정한다.
2. 초기 등록 대상 인플루언서 20명을 선정한다.
3. 인플루언서, 채널, 라이브 상태, 콘텐츠 데이터 구조를 확정한다.
4. 홈과 상세 페이지를 실제 UI 수준으로 한 단계 더 구체화한다.
5. 수동 운영이 필요하면 간단한 어드민 또는 시트 운영안을 만든다.

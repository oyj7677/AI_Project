# Summary

`MyStarNow`를 "인플루언서의 여러 플랫폼 소식을 모아보는 서비스"에서 "아이돌 그룹별 공식 유튜브와 멤버 개인 유튜브를 한곳에서 모아보는 서비스"로 재정의한다. 사용자는 특정 그룹의 공식 채널, 멤버별 개인 채널, 최근 업로드 영상을 한 번에 확인하고, 여러 그룹을 그룹 단위로 탐색할 수 있어야 한다. 1차는 `YouTube`만 지원하며, 핵심 가치는 "아이돌 그룹 중심으로 흩어진 유튜브 채널을 정리해주는 발견성과 편의성"이다.

# Problem

- 아이돌 팬은 그룹 공식 채널과 멤버별 개인 채널을 각각 따로 구독하고 찾아가야 한다.
- 멤버 개인 유튜브가 따로 있는 경우, 어떤 채널이 공식 채널인지, 어떤 채널이 멤버 개인 채널인지 빠르게 파악하기 어렵다.
- 특정 그룹의 최근 유튜브 활동을 보려면 여러 채널을 직접 오가며 확인해야 한다.
- 여러 그룹을 함께 챙기는 팬 입장에서는 "그룹 단위로 묶인 유튜브 허브"가 없어 탐색 비용이 크다.
- 신규 팬은 그룹 입문 시 공식 채널과 멤버 개인 채널 구조를 이해하기 어렵다.

# Goals

- 아이돌 그룹별 공식 유튜브 채널과 멤버 개인 유튜브 채널을 한곳에서 묶어 보여준다.
- 그룹 중심으로 최근 업로드 영상을 시간순으로 보여준다.
- 사용자가 그룹 단위로 빠르게 탐색하고, 멤버 개인 채널까지 쉽게 진입할 수 있게 한다.
- 공식 채널과 멤버 개인 채널의 관계를 명확하게 표현한다.
- 1차 범위는 `YouTube` 하나에 집중해 데이터 정확도와 탐색 경험을 먼저 완성한다.
- 사용자가 특정 그룹의 최근 유튜브 활동을 10초 안에 파악할 수 있게 한다.

# Non-Goals

- 1차 범위에서 Instagram, X, TikTok, Weverse, Bubble 같은 타 플랫폼은 다루지 않는다.
- 1차 범위에서 자체 영상 재생, 댓글, 좋아요, 알림, 커뮤니티 기능은 만들지 않는다.
- 1차 범위에서 모든 아이돌 데이터를 자동으로 완전 수집하는 대규모 크롤링 서비스는 목표로 하지 않는다.
- 1차 범위에서 팬덤 커뮤니티, 굿즈, 스케줄 캘린더 중심 서비스로 확장하지 않는다.
- 1차 범위에서 음원, 뮤직비디오, 공연 정보까지 제품 범위를 넓히지 않는다.

# Product Concept

## One-line Definition

아이돌 그룹별 공식 유튜브와 멤버 개인 유튜브를 한곳에서 모아보는 그룹 중심 유튜브 허브

## Core Value Proposition

- 그룹 단위로 채널 구조를 정리해준다.
- 공식 채널과 멤버 개인 채널을 한 번에 보여준다.
- 여러 채널의 최근 영상을 통합 피드로 보여준다.
- 여러 그룹을 동시에 관리하기 쉬운 탐색 구조를 제공한다.

## Why Now

- 아이돌 멤버들의 개인 유튜브 개설이 늘어나면서 "그룹-멤버-채널" 구조를 정리해주는 서비스 가치가 커졌다.
- 1차를 YouTube 하나로 좁히면 데이터 구조와 사용자 흐름을 더 명확하게 검증할 수 있다.
- 현재 구현된 BFF, 관리자 입력 구조를 활용하면 방향 전환 비용이 비교적 작다.

# Users / Scenarios

## Primary Users

- 특정 아이돌 그룹을 꾸준히 챙겨보는 팬
- 여러 그룹을 동시에 구독하는 멀티팬
- 새로 입덕해서 공식 채널과 멤버 개인 채널 구성을 알고 싶은 사용자

## Secondary Users

- 팬 계정 운영자
- 그룹별 영상 업데이트를 추적하는 큐레이션 운영자

## Core Scenarios

### Scenario 1. 그룹 단위 탐색

- 사용자가 홈에서 여러 아이돌 그룹을 본다.
- 각 그룹 카드에는 그룹명, 공식 채널 수, 멤버 개인 채널 수, 최근 업로드 시간이 보인다.
- 사용자는 관심 있는 그룹을 눌러 그룹 상세로 진입한다.

### Scenario 2. 그룹 상세에서 채널 구조 이해

- 사용자가 그룹 상세 페이지에 들어간다.
- 서비스는 `공식 채널`, `멤버 개인 채널`, `최근 업로드 피드`를 분리해 보여준다.
- 사용자는 어떤 채널이 그룹 공식 채널인지, 어떤 채널이 멤버 개인 채널인지 바로 이해한다.

### Scenario 3. 그룹 통합 피드 소비

- 사용자가 특정 그룹의 최근 영상을 본다.
- 서비스는 공식 채널 영상과 멤버 개인 영상에 배지를 붙여 시간순으로 보여준다.
- 사용자는 더보기 버튼으로 다음 영상 묶음을 이어서 본다.

### Scenario 4. 멤버별 채널 발견

- 사용자가 그룹 상세에서 멤버 리스트를 본다.
- 멤버 카드에는 이름, 개인 채널 보유 여부, 최근 업로드 시간이 나온다.
- 사용자는 특정 멤버의 개인 채널로 바로 이동한다.

### Scenario 5. 여러 그룹 비교

- 사용자가 그룹 목록에서 최근 활동순으로 정렬한다.
- 어떤 그룹이 최근 유튜브 활동이 많은지 빠르게 파악한다.

# Domain Model Concept

## Core Entities

- `Group`
  - 그룹명
  - 슬러그
  - 소개
  - 대표 이미지
  - 활동 상태
- `Member`
  - 이름
  - 슬러그
  - 그룹 소속
  - 프로필 이미지
  - 개인 채널 보유 여부
- `Channel`
  - 플랫폼: `YouTube`
  - 채널 유형: `GROUP_OFFICIAL`, `MEMBER_PERSONAL`, `SUB_UNIT`, `LABEL`
  - 소유 주체: 그룹 또는 멤버
  - 핸들
  - 채널 URL
  - 공식 여부
- `Video`
  - 영상 제목
  - 썸네일
  - 게시 시각
  - 영상 URL
  - 업로드 채널
  - 그룹/멤버 연결 정보

## Important Rules

- 하나의 그룹은 여러 공식 채널을 가질 수 있다.
- 하나의 그룹은 여러 멤버를 가진다.
- 멤버는 0개 또는 1개 이상의 개인 채널을 가질 수 있다.
- 홈/그룹 피드는 채널 중심이 아니라 `그룹 중심`으로 재정렬한다.
- 영상에는 반드시 "어느 그룹의 어떤 채널에서 올라온 영상인지"가 표시되어야 한다.

# Feature Proposal

## 1. 그룹 디렉터리

설명:
아이돌 그룹을 그룹 단위로 탐색하는 진입 기능이다.

포함 정보:

- 그룹명
- 대표 이미지
- 한 줄 소개
- 공식 채널 수
- 멤버 개인 채널 수
- 최근 영상 게시 시각

예시:

- `StarWave | 공식 2개 · 멤버 개인 4개 · 최근 업로드 2시간 전`

## 2. 그룹 헤더

설명:
그룹 상세 상단에서 그룹의 정체성과 채널 구성을 요약한다.

포함 정보:

- 그룹명
- 소개
- 멤버 수
- 공식 채널 수
- 멤버 개인 채널 수
- 최근 업로드 시각

## 3. 공식 채널 허브

설명:
그룹 공식 채널을 한 번에 정리해서 보여준다.

채널 카드 예시:

- 채널명
- 핸들
- 채널 유형
- 최근 업로드 시각
- 바로가기 버튼

예시 유형:

- 그룹 공식
- 서브 유닛
- 레이블 운영

## 4. 멤버 개인 채널 섹션

설명:
멤버별 개인 유튜브 채널을 그룹 상세 안에서 찾기 쉽게 보여준다.

멤버 카드 예시:

- 멤버명
- 개인 채널명
- 최근 업로드 시각
- 개인 채널 바로가기

## 5. 그룹 통합 영상 피드

설명:
그룹 공식 채널과 멤버 개인 채널의 최근 업로드를 한 타임라인으로 보여준다.

피드 카드 필수 정보:

- 썸네일
- 영상 제목
- 업로드 채널명
- 그룹 또는 멤버 배지
- 채널 유형 배지
- 게시 시각
- 유튜브 바로가기

동작:

- 기본 8개 노출
- `더보기`로 다음 8개 로드
- 정렬 기준은 최신순

## 6. 멤버 상세

설명:
특정 멤버를 눌렀을 때 개인 채널 중심으로 보는 보조 화면이다.

포함 정보:

- 멤버명
- 소속 그룹
- 개인 채널 목록
- 최근 영상 목록

## 7. 즐겨찾기

설명:
자주 보는 그룹을 저장해 빠르게 접근한다.

MVP 방식:

- 로그인 없이 로컬 저장

## 8. 운영자 입력 기능

설명:
공식 채널과 멤버 채널, 그룹-멤버 연결 정보를 보정하기 위한 내부 운영 기능이다.

운영 대상:

- 그룹 생성
- 멤버 생성
- 채널 생성
- 채널 유형 수정
- 그룹/멤버 연결 수정
- 수동 영상 등록

# Example Service Flow

- 그룹명: `StarWave`
- 공식 채널:
  - `StarWave Official`
  - `StarWave Behind`

  - `하린`

  - `하린 로그`
  - `시우 다이어리`

- 홈에서 `StarWave` 그룹을 발견한다.
- 그룹 상세에 들어가 공식 채널 2개와 멤버 개인 채널 2개를 본다.
- 최근 업로드 피드에서 공식 콘텐츠와 멤버 브이로그가 섞여 나온다.
- 사용자는 "그룹 전체 유튜브 활동"을 한 화면에서 이해한다.

## Deferred

- Instagram 등 타 플랫폼 연동
- 푸시 알림
- 추천 알고리즘
- 재생 목록 큐레이션
- 커뮤니티 기능

# Data Strategy

## Phase 1

- YouTube Data API v3 사용
- 그룹 공식 채널과 멤버 개인 채널을 수동으로 등록
- 영상 목록과 채널 메타는 자동 수집
- 그룹-멤버-채널 관계는 운영자가 관리

## Why This Strategy

- YouTube는 공식 API로 안정적으로 채널/영상 정보를 가져오기 쉽다.
- 핵심 난제는 "채널 간 관계를 묶는 것"이지 영상 수집 자체가 아니다.
- 따라서 MVP에서는 관계 데이터는 수동 관리, 영상 수집은 자동화가 가장 현실적이다.

# Success Metrics

- 사용자가 그룹 상세에서 공식 채널과 멤버 개인 채널 구조를 5초 안에 이해할 수 있다.
- 홈 피드에서 영상 클릭 전환율이 20% 이상이다.
- 그룹 상세 진입 후 이탈 없이 2개 이상 채널을 탐색하는 비율이 40% 이상이다.
- 그룹 등록 후 1시간 이내 최신 영상이 정상 반영된다.
- 운영자가 신규 그룹 온보딩을 10분 이내 완료할 수 있다.

# Acceptance Criteria

- [ ] 사용자는 그룹 목록에서 여러 아이돌 그룹을 카드 형태로 탐색할 수 있다.
- [ ] 사용자는 그룹 상세에서 공식 채널과 멤버 개인 채널을 구분해서 볼 수 있다.
- [ ] 사용자는 그룹 통합 최근 업로드 피드를 최신순으로 볼 수 있다.
- [ ] 사용자는 더보기 버튼으로 다음 영상 묶음을 불러올 수 있다.
- [ ] 영상 카드에는 그룹/멤버/채널 유형이 명확하게 표시된다.
- [ ] 운영자는 그룹, 멤버, 채널 관계를 입력 및 수정할 수 있다.
- [ ] 1차 데이터 소스는 YouTube만 사용한다.

# Risks

- 아이돌 그룹과 멤버 개인 채널의 관계 데이터는 자동 판별이 어렵다.
- 공식 채널과 비공식 채널을 잘못 연결하면 신뢰도가 크게 떨어진다.
- 멤버 개인 채널 여부 판단 기준이 모호할 수 있다.
- 그룹/멤버/서브유닛 구조가 복잡한 경우 데이터 모델이 흔들릴 수 있다.
- 특정 그룹의 채널 수가 많아지면 UI가 복잡해질 수 있다.

# Open Questions

- 그룹과 멤버를 모두 1차 MVP에서 노출할지, 그룹 중심 화면만 먼저 갈지
- 레이블 운영 채널을 공식 채널과 같은 레벨로 볼지
- 멤버 개인 채널이 아닌 브이로그/예능용 별도 채널을 어떻게 분류할지
- 그룹 대표 이미지와 멤버 대표 이미지를 어디서 가져올지
- 그룹별 즐겨찾기 외에 멤버 단위 즐겨찾기도 필요한지
